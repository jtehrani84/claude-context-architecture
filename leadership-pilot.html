The math: If each SE saves 45 min/day on account prep, meeting prep, email drafting, and content generation — that's 15+ hours/month back per person. Across a team of 60, that's 900 hours/month of SE capacity recovered.

The honest framing

Use Slack AI when you need to...

  • Find something said in a channel last week
  • Summarize a long thread you missed
  • Quick answer from company knowledge base
  • Share an insight with the team in-channel

Use Claude Code harness when you need to...

  • Prep for a customer meeting with full intelligence
  • Generate a competitive talk track that sounds like you
  • Build a demo, prototype, or technical artifact
  • Run a multi-step research → analysis → deliverable pipeline
  • Have the system learn from your corrections permanently

Timeline reality: Slack AI is a great channel search tool today. It may add memory and skills depth over the next 12-18 months. The local harness delivers that value now — and compounds daily. Starting today means your team has a 12-month head start on institutional knowledge that no centralized tool can replicate, because it encodes your judgment, your corrections, your org structure.

2-Week Leadership Pilot

1
Day 1: Install & Setup (30 min)

Clone starter kit. Run setup script. Verify CLAUDE.md loads, rules fire, first skill (/account-prep) works. Done in one terminal session.

2
Days 2-3: Core Workflows (normal work)

Use Claude Code for your actual work. Try /account-prep before a meeting. Try /email-draft for a customer email. Notice how rules prevent AI-sounding output automatically.

3
Days 4-7: Memory Compounds

Correct Claude when it gets something wrong. Watch it never make that mistake again. Add your accounts, your team, your preferences. The system learns YOUR judgment.

4
Days 8-10: Intelligence Layer

Activate overnight crons. Check wiki/inbox.md each morning for auto-gathered intelligence. Seed entity pages for your direct reports. The system starts working while you sleep.

5
Days 11-14: Evaluate & Decide

Compare output quality vs. pre-pilot. Measure time saved. Assess: does this warrant org-wide rollout? What would you change for your team?

What pilot participants need

✓ Claude Code installed (VS Code extension or CLI) — already deployed via LLMGW
✓ Access to starter kit repo (private GitHub, @salesforce.com auth)
✓ 30 minutes for initial setup (one time)
✓ Willingness to use Claude Code for 2-3 real work tasks per day during pilot
✓ Brief weekly check-in (15 min) to share findings

Success Criteria — What "Pass" Looks Like

1. Time-to-First-Value

Participant can run /account-prep on a real account and get useful output within 48 hours of setup. No hand-holding required after initial install.

Target: < 48 hours from clone to first useful output

2. Output Quality Delta

Customer-facing content (emails, talk tracks, prep docs) produced with the harness is demonstrably higher quality than without. No AI-sounding language. Correct product names. Customer-centric framing.

Target: Participant would send the output to a customer without editing

3. Compound Learning Demonstrated

By Day 7, Claude remembers corrections from Day 2. Participant sees the system getting better without re-explaining. Memory accumulates visibly.

Target: 10+ memory files created organically by end of Week 1

4. Workflow Acceleration

At least one workflow the participant does weekly now takes <50% of the time it used to. Account prep, meeting prep, email drafting, competitive research — any one.

Target: 1+ workflow at 50% time reduction

5. "Would Recommend" Signal

After 2 weeks, participant answers: "Would you want your team to have this?" If yes — pilot passes. If no — we learn why and iterate.

Target: Unanimous "yes" from pilot cohort

What we're NOT measuring

  • ✗ Lines of code written (irrelevant for SE leadership)
  • ✗ Number of sessions per day (usage ≠ value)
  • ✗ Technical complexity of setup (we handle that)
  • ✗ Comparison to Slack AI (different tool, different job)

The real question: Does having a context-engineered Claude Code setup make an SE measurably better at their job? Not "faster at typing" — better at the work that wins deals.
